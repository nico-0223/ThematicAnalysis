# Trustworthiness and audit trail

The application supports transparent qualitative analysis through audit trail records, memos, reflexive journal entries, coding-decision logs, codebook versioning, explicit inclusion and exclusion criteria, representative evidence, adjudication records, and exportable methods sections.

Audit trail entries are created when codebooks are loaded, conversations are ingested, segments are produced, analysis runs are started, phase status is updated, annotations are created, candidate themes are generated, candidate themes are reviewed, reliability is calculated, and reports are exported.

Memos can be associated with conversations, segments, coders, and phases. Reflexive journal entries can be associated with analysis runs and phases. These records make analytic assumptions and decisions easier to inspect.

Codebook versioning allows the analyst to preserve which definitions, indicators, and criteria were used for a given run.

Coding-decision logs preserve why an annotation was created, imported, or manually added.

Adjudication records preserve decisions about unresolved disagreements.

Automated coding is limited. Rule-based coding only applies editable indicators from the codebook. It may support organisation and retrieval, but it does not replace interpretation.

Inter-coder agreement is optional. It is useful when the chosen analytic orientation treats comparable coder assignments as relevant. It is not mandatory for every form of thematic analysis.
