# Theoretical framework

The application is grounded in Braun and Clarke's six-phase thematic analysis. The phases are used to structure the workflow rather than simply label outputs.

Phase 1, familiarisation, is represented through data preservation, conservative preprocessing, and memo records. Phase 2, generating initial codes, is represented through manual annotations, imported annotations, or rule-based annotations created from editable indicators. Phase 3, searching for themes, aggregates coded segments into candidate themes. Phase 4, reviewing themes, checks support for candidate themes and records review decisions. Phase 5, defining and naming themes, is supported through analytical definitions, inclusion criteria, exclusion criteria, examples, and counterexamples in the codebook. Phase 6, producing the report, exports the method, phase status, codebook version, evidence, audit trail, memos, limitations, and references.

The software does not determine final themes. It records the methodological structure and makes analytic decisions inspectable. Substantive interpretation remains the responsibility of the analyst.

The methodological framework is distinct from substantive interpretation. The framework specifies how analysis is organised; the analyst specifies what the dataset means, what theory is relevant, what counts as evidence, and which themes are finally accepted.

References:

Braun, V., & Clarke, V. (2006). Using thematic analysis in psychology. *Qualitative Research in Psychology, 3*, 77–101.

Nowell, L. S., Norris, J. M., White, D. E., & Moules, N. J. (2017). Thematic Analysis: Striving to Meet the Trustworthiness Criteria. *International Journal of Qualitative Methods, 16*, 1–13.

Boyatzis, R. E. (1998). *Transforming Qualitative Information: Thematic Analysis and Code Development*. Sage.
